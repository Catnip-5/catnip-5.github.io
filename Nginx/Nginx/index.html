内置变量

  • 客户端请求相关变量：

    • $host 客户端请求的 Host 头值（若无则取服务器名称或 IP）。
    • $remote_addr 客户端的 IP 地址。
    • $remote_port 客户端连接的端口号。
    • $request 完整的请求行（如 GET /index.html HTTP/1.1）。
    • $request_method 请求方法（如 GET、POST）。
    • $request_uri 原始请求 URI，包括参数（如 /index.html?a=1）。
    • $uri 当前请求的 URI，不包含参数（如 /index.html）。
    • $args 请求的查询字符串参数（如 a=1&b=2）。
    • $arg_xxx 特定查询参数的值（如 $arg_a 为 1）。
    • $http_xxx 客户端请求的 HTTP 头字段（xxx 为头名小写，如 $http_user_agent 表示 User-Agent）。
    • $scheme 请求使用的协议（如 http 或 https）。

  • 服务器相关变量：

    • $server_addr 服务器的 IP 地址（需启用 set_real_ip_from 或动态解析）。
    • $server_port 服务器监听的端口。
    • $server_name 当前匹配的 server_name。
    • $server_protocol 请求使用的协议版本（如 HTTP/1.1）。

  • 连接和请求处理相关变量：

    • $connection 当前连接的唯一 ID。
    • $connection_requests 当前连接处理的请求总数。
    • $request_time 处理请求的总时间（以秒为单位，精确到毫秒）。
    • $upstream_response_time 从上游服务器接收响应的时间。
    • $upstream_addr 上游服务器的地址（IP:端口）。
    • $upstream_status 上游服务器的响应状态码。
    • $bytes_sent 发送给客户端的字节数。
    • $body_bytes_sent 发送给客户端的响应体字节数（不包括头）。

  • 文件和路径相关变量：

    • $document_root 当前请求的 root 指令值。
    • $document_uri 当前请求的 URI，经过内部重定向后（如 alias 或 rewrite 后的值）。
    • $request_filename 当前请求的文件路径（结合 root 或 alias 解析）。
    • $fastcgi_script_name FastCGI 请求的文件名。

  • 状态和条件变量：

    • $status 响应状态码（如 200、404）。
    • $sent_http_xxx 发送给客户端的 HTTP 头字段（如 $sent_http_content_type）。
    • $is_args 如果请求有查询字符串，则为 ?，否则为空。

  • 其他高级变量：

    • $pid 当前工作进程的 ID。
    • $msec 当前时间（以 Unix 时间戳+毫秒格式，如 1623456789.123）。
    • $nginx_version NGINX 版本号。
    • $time_local 本地时间（格式如 [12/Oct/2023:12:00:00 +0800]）。
    • $cookie_xxx 特定 cookie 值（如 $cookie_sessionid）。

location 匹配规则

  • 匹配优先级：
    • 精确匹配 (=): 最高优先级。
    • 前缀优先匹配 (^~): 次高优先级，阻止正则匹配。
    • 前缀匹配 (无修饰符): 按最长匹配原则选择。
    • 正则匹配 (~ 和 ~*): 按配置文件中定义的顺序依次尝试。
    • 通用匹配 (/): 兜底，优先级最低。

rewrite 重写URL

  • rewrite语法： rewrite regex replacement [flag];

    • regex 正则表达式，匹配请求的 URI。
    • replacement 重写后的目标 URI。
    • flag（可选） 控制重写行为，常见标志包括：
      • last 完成重写后停止处理当前 location，重新启动请求匹配。
      • break 完成重写后停止处理，留在当前 location。
      • redirect 返回 302 临时重定向（客户端感知）。
      • permanent 返回 301 永久重定向（客户端感知）。

valid_referers 防盗链

  • valid_referers语法： valid_referers [none] [blocked] [server_names] [string ...];
    • none 允许没有 Referer 头的请求（例如直接在浏览器输入 URL）。
    • blocked 允许 Referer 被防火墙或代理过滤的请求。
    • server_names 允许来自当前服务器域名的请求。
    • string 具体域名：example.com; 子域匹配：*.example.com; 正则表达式：~^(https?://)?(.*\.)?example\.com。
